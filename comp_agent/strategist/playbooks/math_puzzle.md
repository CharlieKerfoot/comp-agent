# Math Puzzle Playbook

## Baseline Phase
1. **Small cases**: Enumerate solutions for small inputs
2. **Pattern recognition**: Look for sequences, symmetries, invariants
3. **Brute force**: Exhaustive search to verify understanding

## Improve Phase
1. **Mathematical structure**: Prove properties to reduce search space
2. **Efficient algorithms**: Replace brute force with polynomial-time
3. **Memoization**: Cache intermediate results
4. **Mathematical libraries**: SymPy for symbolic computation

## Ensemble Phase
1. **Multiple approaches**: Verify solution via independent methods
2. **Stress testing**: Random test cases to validate correctness

## Polish Phase
1. **Edge cases**: Boundary conditions, special inputs
2. **Implementation optimization**: Bit manipulation, efficient data structures
3. **Proof of correctness**: Formal argument for why solution works
